# SAAI 뉴스레터 자동화 프로젝트

## 목표
트렌드 키워드 입력 + 기상청 날씨 데이터 + 이벤트 파일 + 오피스 상권 매출 데이터를 읽어
편의점 점주 대상 뉴스레터 HTML 파일을 자동 생성한다.

---

## 출력 포맷

> **⚠ 뉴스레터 HTML을 생성·수정할 때는 반드시 `newsletter-html-rules` skill을 사용할 것.**
> 프로젝트 로컬 skill(`.claude/skills/newsletter-html-rules/`)이 단일 기준이며, id 매핑·섹션 주석·템플릿 구조는 skill 지침을 따른다.

최종 발송용 출력은 **stibee 포맷**:
- `<table>` 기반 레이아웃 (이메일 클라이언트 호환)
- 모든 CSS를 `style=""` 인라인으로 작성
- **HTML 구조·id 속성·섹션 주석 포맷은 `newsletter-html-rules` skill을 반드시 따를 것**
  - `report-poll.py`가 id로 파싱 (header-title, section-title, card, card-title, card-body, card-badge, weather-*, event-*)
  - 섹션 주석: `<!-- ── MODULE X: 제목 ── -->` ~ `<!-- /MODULE X -->`
  - canonical 템플릿: skill 디렉토리의 `newsletter-template.html`
- 피드백 버튼에 Google Forms 링크 연결
- 푸터에 "매출 경향은 내부 분석 기반으로, 상권에 따라 다를 수 있어요" 면책 문구 포함

실제 발송 파일은 `outputs/newsletter-YYYY-MM-DD-stibee.html`.

## 실행 프로세스

### 1단계: 키워드 수집
```
뉴스레터 생성해줘
트렌드: 버터떡, 두쫀쿠, 비빔면 제로슈거, ...
```
사용자가 X(트위터) 데이터 테이블을 제공하면:
- 중복 키워드 제거
- RT·Likes 수치 추출 (트렌드 규모 판단 참고)

### 2단계: 키워드 트렌드 확인
각 키워드의 검색 변형을 판단한 뒤 `fetch_naver_trend.py` 실행.
콜론(:)으로 합산 그룹 지정 가능 (예: `"진밀면:오뚜기 진밀면, 두쫀쿠"`).
상세 절차는 `prompt/newsletter-prompt.md`의 "키워드 트렌드 조회 절차" 참조.

**데이터랩 활용 — 두 가지 용도를 구분할 것:**

1. **트렌드 카테고리 키워드** (버터떡, 두쫀쿠, 우베, 단백질쉐이크 등)
   - 떡볶이 기준선 비교로 규모 판단 → 배지(HOT/주목/초기) 표시
   - HOT (30%+) → 초록 pill 배지
   - 주목 (10~29%) → 노란 pill 배지
   - 초기 (3~9%) → 노란 pill 배지
   - 3% 미만 → 배지 없음
   - 주간 변화율은 본문에 자연스럽게 녹이기 (배지에 숫자 넣지 말 것)

2. **개별 상품명** (박은영 마라샹궈, 정호영 다카마쓰우동, 에드워드리 김치전 등)
   - 떡볶이 비교 의미 없음 — 특정 상품명 검색량은 카테고리보다 훨씬 작음
   - **배지 달지 말 것** — 대신 기사 팩트(매출 OO만개, 카테고리 1위)와 SNS 반응(RT, 품절 언급)으로 설명
   - 합산 키워드 주의: "마라샹궈"(카테고리)와 "박은영 마라샹궈"(상품)를 합치면 카테고리 검색량이 오염됨

### 키워드 편집 판단 기준
- **"이번 주에 새로 터졌거나 변화가 있는 것"만 넣기** — 주간 변화율로 판단
  - +50% 이상: 이번 주 폭발 → 넣기
  - +20~50%: 이번 주 상승 → 새 소식 있으면 넣기
  - ±20% 이내: 유지 → 이번 주 새 소식 없으면 빼기 (스테디셀러 제외)
- **출시일이 1개월 이상 지난 상품은 이번 주 트렌드로 넣지 말 것**
- **편의점에서 살 수 있으면 편의점 트렌드** — 완제품이 아니어도 재료를 편의점에서 사면 해당
- **편의점 미입점 상품은 메가트렌드** — "순차 출시 예정"도 아직 입점 아님
- **아이돌/콘서트/컴백은 이벤트가 아니라 메가트렌드** — 이벤트 섹션은 캘린더 이벤트만 (블랙데이, 근로자의 날 등)
- **모르는 건 아는 척 하지 말고 빼기** — 확인 안 된 SNS 수치, 출처 불명확한 팩트 절대 금지
- **편집 판단에 대한 설명을 뉴스레터에 넣지 말 것** — 점주에겐 결과만

### 3단계: 키워드 분류 (웹 검색)
각 키워드를 편의점 입점 여부로 분류:
- 편의점 입점/신상 확인 → **편의점 트렌드** (트렌드 카테고리만 배지, 개별 상품은 팩트로)
- 편의점 입점 없음 → **메가트렌드** (배지 없음)

유사 키워드는 하나로 묶을 수 있음:
- 예: 세븐일레븐×롯데자이언츠 + SSG랜더스×이마트24 → "KBO 개막 콜라보"

### 4단계: 날씨·이벤트 데이터
- `fetch_weather.py` 실행 → `data/weather.json` 생성
- `data/events.md` 읽어서 한 달 이내 이벤트 확인
- 날씨 뉴스 검색 (꽃샘추위, 황사, 미세먼지, 건조주의보, 이른 더위 등)

### 5단계: 상권 데이터 반영
`가설검증_통합결과.xlsx` 의 오피스 상권 매출 가설검증 결과를 참고하여:
- 날씨 판매 TIP에 데이터 근거 반영 (예: "15°C 넘으면 냉음료가 눈에 띄게 늘어요")
- 이벤트 팁에 공휴일/주말 매출 경향 반영 (예: "공휴일에 방문 고객이 크게 줄지만 1인당 구매 금액은 올라가는 경향이 있어요")
- **주의: 구체적 퍼센트 숫자를 직접 노출하지 않고 자연스럽게 녹여서 표현**

### 6단계: HTML 생성
- 출처 검색 → 공신력 있는 기사 링크 확보 (기사 날짜 함께 표기)
- 모든 가이드 파일 참조하여 HTML 작성
- `outputs/newsletter-YYYY-MM-DD-stibee.html` 생성

**HTML 작성 시 반드시 지킬 것:**
- **가격 넣지 말 것**: 점주는 가격을 이미 알고 있음. 행사 정보(2+1 등)는 넣되 가격 자체는 불필요
- **떡볶이 비교 노출 금지**: 떡볶이 기준선은 내부 판단 도구. 점주에게 "떡볶이의 N%" 같은 표현 절대 쓰지 말 것
- **검증 안 된 숫자 금지**: 모든 숫자는 기사 URL을 직접 열어서 확인한 것만 기재. 에이전트/검색 요약 맹신 금지
- **영어 라벨 금지**: CHEF COLLAB 같은 영어 쓰지 말 것. 점주 독자층은 30~40대
- **상권 태그 별도 줄 금지**: 🏪 태그를 따로 빼지 말고 본문 설명에 자연스럽게 녹이기
- **독점 표기**: 기사에서 "독점"/"단독" 확인된 것만. 아니면 편의점명만 쓰고 통일
- **헤드라인 고정**: "N월 N주차 스토어레터" 형식. 임의로 후킹 제목 만들지 말 것
- **날씨 권역별 6개 필수**: 수도권/강원/충청권/전라권/경상권/제주. "전국"으로 뭉치지 말 것
- **출처는 최근 기사만**: 1~2주 이내 기사. 1개월+ 지난 기사 달지 말 것
- **전체 리뷰 필수**: 작성 후 처음부터 끝까지 점주 입장에서 통독. 부분 수정 반복하지 말고 전체를 한 번에 검수

## 주의사항

- HTML 구조·id·style은 `newsletter-html-rules` skill이 단일 기준. prompt에는 콘텐츠·톤 규칙만
- 트렌드 상태값은 반드시 네이버 DataLab + 웹 검색으로 판단. 추측 금지.
  - 트렌드 카테고리 → 데이터랩 기준선 비교 + 배지
  - 개별 상품명 → 데이터랩 비교 의미 없음, 기사 팩트 + SNS 반응으로 판단
- 합산 키워드 오염 주의: 카테고리명(마라샹궈)과 상품명(박은영 마라샹궈)을 합치면 안 됨
- 편의점 신상 제품명은 검색 확인된 경우에만 기재. 출시일 1개월+ 지난 상품은 "이번 주 트렌드"로 넣지 말 것.
- 출처 링크는 공신력 있는 언론사·공식 보도자료만 사용. 최근 1~2주 이내 기사만. 기사 날짜 함께 표기.
- 기사의 숫자(매출, 판매량 등)는 기사 URL을 직접 열어서 확인한 것만 인용. 에이전트나 검색 요약을 그대로 믿지 말 것.
- 모르는 건 아는 척 하지 말고 빼기. "~다고 합니다" 같은 출처 불명확 표현 금지.
- 출처를 못 찾은 키워드도 작성 가능 (출처 링크만 생략).
- 이벤트 요일은 반드시 확인.
- 상권 데이터 수치는 직접 노출 금지 → "경향이 있어요", "눈에 띄게 늘어요" 등 자연어로 표현.
- `.env` 파일 절대 Git에 올리지 말 것.
- `env.env`는 구버전 환경변수 파일 (무시).
