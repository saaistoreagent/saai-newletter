## 트렌드 점수 & HOT 배지 (★ 단일 룰)

**점수 = (떡볶이 대비 절대 검색량 %) × (폭증률 배수)**
- 절대 검색량: 떡볶이를 모든 데이터랩 배치에 앵커로 넣어 정규화 (떡볶이 = 100%)
- 폭증률: 이번주 평균 ÷ 전주 평균 - 1 (마이너스는 0)
- HOT 기준: 점수 ≥ 50
- 카드 정렬: 점수 높은 순

**Why**: 절대량만 보면 스테디셀러가 위로 가서 신선한 신호 약해짐. 폭증률만 보면 작은 신호 노이즈. 둘 다 큰 키워드가 진짜 점주 가치 큰 트렌드.

**How to apply**: ▲/—/▼ 배지 안 쓰고 HOT 단일. 떡볶이 비교는 내부용 (점주에게 노출 금지).

**섹션 헤더 정렬 안내**: 카드 목록 위에 "검색량·SNS 반응 강한 순" 한 줄만 (우상단 회색 12px). 구체 계산식 노출 금지.

**금지**:

**Why**: 2026/04/22 세션에서 루리웹 링크, Threads 링크로 출처 만들었다가 신뢰도 문제로 전부 교체해야 했음.

**How to apply**: 카드 작성 전 **반드시 언론사 기사 먼저 검색**. 기사 못 찾으면 출처 링크 생략(본문은 작성 가능).

## 할루시네이션 방지 (★ 핵심)

### 절대 금지 패턴
- **GPT 요약·Grok 요약·SNS 요약을 검증 없이 사용** — 기사 URL 직접 fetch해서 원문 확인해야
- **가격·출시일·상품명을 추측으로 기재**
- **"본격 출시 X월 X일" 같은 일정을 검색 요약만 보고 단정**
- **검색 못 했으면 생략. 지어내지 말 것**

### 주관적 판단 금지
- ❌ "~찾는 고객이 많아요" (추측)
- ❌ "10~20대가 찾아요" (인구통계 추측)
- ❌ "특수가 크다/작다" (데이터 없이 단정)
- ❌ "호불호가 적다" (주관)
- ❌ "수요가 붙어요" (업계 jargon)

### 검증 과정
1. 제품명·가격·출시일 = **언론사 기사 URL 직접 fetch**
2. 콜라보 상품 리스트 = **공식 발표 원문 확인**
3. 검색량 주장 = **데이터랩 실측과 대조**

**Why**: 2026/04/22 세션에서 "본격 출시 4/29", "한상가득도시락 5,900원", "스타벅스 우베 라떼" 전부 제가 지어낸 것. 사용자가 여러 번 지적.

## SNS 애칭 처리

- SNS 애칭 발견 시 정식 상품명 반드시 검색 (펴늬·블로그·인스타·커뮤니티 다 활용)
- 안 잡히면 사용자에게 정식 명칭 확인 요청. **임의 매핑 금지**
- 본문에 "(SNS 애칭 'XX')" 명시

**SNS 애칭 ↔ 정식 명칭 사례 (반복 활용 가능)**
- 두쫀쿠 = 두바이 쫀득 쿠키 (편의점별 라인업)
- 황쫀쿠 = CU 단짠 황치즈 쫀득쿠키
- 콘피쵸 = 크라운 콘쵸 + 피스타치오 한정판
- 니모 아이스크림 = GS25 프로즌 피쉬 소르베
- 전남친토스트맛 = 빵또아 블루베리 크림치즈맛 (빙그레×끼리)
- 깨먹는 하트 티라미수 = 실제 디저트 브랜드명 (연세우유 생크림빵 콜라보)

## 카드 표기 형식 (★ 핵심)

- **편의점 한정** → 카드 우측 상단에 **브랜드 색상 텍스트** 표기 (pill·박스·stripe 금지)
  - 7-ELEVEN: `#EE162E`(7) + `#046A38`(-ELEVEN) 분리
  - GS25: `#003B7B`
  - CU: `#552084`
- **편의점 공용** (일반 제조사 상품): `ALL` 회색 `#787f89`
- "X" 콜라보 표기 금지 → "·"로 통일

**Why**: 점주가 자기 매장에 의미 있는 상품인지 즉시 판단. CU 점주에게 GS25 단독 상품 정보는 무의미.

## 데이터랩 사용

- **단독 키워드 일별(date) 조회만** — 주간 / 5개 묶음 비교 금지
- 떡볶이 앵커 + 5개씩 묶어서 절대량 정규화
- 점주에게는 떡볶이 비교 노출 금지 (내부용)
- 제품명이 길어 검색 안 잡히는 경우 시리즈/카테고리 키워드로 재조회 (예: "깨먹는 하트 생크림빵" 0.2% → "연세우유 생크림빵" 5.4%)

## 이벤트 섹션 (MODULE 5A)

### 범위
- **발행일 ~ 한 달 이내만** 포함. 예: 발행 4/22 → 5/22까지만

### 정식 명칭
- **근로자의 날 = "법정휴일"** (유급휴일·근로기준법). "준공휴일" 아님
- 어린이날·설날·추석 등 = "법정공휴일"
- events.md 원본에 오류 있을 수 있음 — **반드시 웹 검색으로 재검증**

### 음력 기반 날짜
- 부처님오신날(석가탄신일): 매년 양력 다름. 웹 검색 필수
- 설·추석 등: 동일

### 이벤트 본문 규칙
- **상권 구분 남발 금지** ("오피스 상권은 줄고, 주택가는 늘고" 반복 나열 X)
- 수요 중심으로 서술
- 편의점 관점에서 의미 있는 정보만. 법적/사회적 배경 설명(예: 김영란법, 근로자의날 법률) 금지
- 라벨에 이미 있는 정보 본문에 중복 금지 (예: 라벨 "법정공휴일·어린이날(화)" + 본문 "법정공휴일(화요일)이에요" 중복)

## 카드 구조

- 카드는 반드시 모듈 섹션 안에 배치 (스크립트 재배치 시 섹션 마커 밖으로 빠지지 않게)
- 모듈 안의 모든 카드는 **auto height** (고정 X)
- 모듈 시작/끝 마커 (`<!-- ── MODULE X -->` ~ `<!-- /MODULE X -->`) 한 쌍씩 정확히
- 카드 제목 영역은 **2컬럼 테이블** (좌: 제품명+배지, 우: 편의점 브랜드)
- 인용블록 스타일: `#f2f2f4` 배경 + rounded-6px + `↳` 화살표. **stripe·border-left 금지**

---

## 디자인 반복 실수 방지

### 절대 금지
- **Border-left stripe** (카드·박스·어디든)
- **카드 배경 컬러 박스 장식**
- **`height:\d+px` 정규식 일괄 제거** — spacer(8px·24px)·피드백 버튼(64px)·이벤트 dot(8px) 전부 망가짐
- **로고 이미지** (크기 제각각 문제 재발)

### dot·버튼 필수 height
- 이벤트 dot: `width:8px;height:8px;` **둘 다 명시**
- 피드백 버튼: `height:64px` + `line-height:64px`
- 카드 간 spacer: `height:8px`
- 모듈 간 spacer: `height:24px`
